Frequently Asked Questions (FAQ)

Thank you for looking at the FAQ.

How do I read .epub files?

On Windows, Mac, and *nix, the Calibre application is recommended for reading and conversions.

For other platforms, please use you favorite search engine to look up "epub <your platform>".

How do I read .mobi and .azw3 files?

These are Amazon Kindle files, which can be read using Kindle devices or the Kindle app on supported platforms.

These files can also be opened by the Calibre application.

How do I edit file descriptions?

File descriptions are stored in ".nfo" files, which are plain text files.

For a file or directory called "foo.bar", create a plain text file named "foo.bar.nfo" and place the description in there.

Why do I need GnuPG?

GnuPG is only needed if you would like to sign your messages and create an identity.

You do not need GnuPG to browse or to post anonymous comments.

How do I install GnuPG?

To install GPG, go to gnupg.org.

How do I create an identity?

You must use a program called GnuPG (aka GPG). It is a command-line program. It is available for your OS. You must learn how to use it. It is not very hard.



To make a new key/identity for yourself, type gpg --gen-key and follow the instructions. You typically only have to do this once. It is like creating an account.

It is not necessary to register on the server before posting a signed message.

If you would like to set a username, just paste your public key into the submission field.

If you get frustrated or stuck, ask a Linux user you know to help you.

How do I sign my messages?

First, you must install GPG and create an identity, as described in the question above.

To sign a new message, first save it to a text file, like SomeFile.txt.

Then type gpg --clearsign SomeFile.txt to sign the contents of that file and save it as SomeFile.txt.asc.

Alternatively, you can type gpg --clearsign and type your message into the command prompt window. Press Ctrl+D (or Ctrl+W on Windows) to finish and display the message, now signed.
